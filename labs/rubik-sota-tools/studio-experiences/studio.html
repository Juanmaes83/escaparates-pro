GitHub Setup

Flujo de trabajo para replicar proyectos de clientes de forma ágil usando GitHub como repositorio de plantillas.

FLUJO COMPLETO — DE RUBIK SOTA A CLIENTE
1
Template master en GitHub
Tienes el repo base rubiksota/experiences-template con la plantilla. Este repo nunca se edita directamente.
2
Por cada cliente nuevo: Fork o desde template
En GitHub, haz fork del template o usa "Use this template". Nómbralo con el slug del cliente: restaurante-sol-torrevieja
3
Edita en RS·STUDIO y exporta config.json
Configura la web del cliente aquí, exporta el config.json y súbelo al repo del cliente en GitHub.
4
Activa GitHub Pages
En el repo del cliente: Settings → Pages → Branch: main → /root. En 2 minutos está online en: cliente.github.io/slug o dominio propio.
5
El cliente edita su propia web (opcional)
Le das la URL con ?edit al final. Cambia textos e imágenes sin código. Descarga su config y te lo manda para actualizar el repo.
COMANDOS RÁPIDOS
Clonar template para nuevo cliente:
gh repo create cliente-nombre --template rubiksota/experiences-template --public
Subir config actualizado:
git add config.json && git commit -m "update config [cliente]" && git push
Activar GitHub Pages (CLI):
gh api repos/{owner}/{repo}/pages -X POST -f source.branch=main -f source.path=/

Publicar en GitHub Pages

Sigue estos pasos para poner la web del cliente online en minutos.

  1. Descarga el config.json usando el botón de abajo o el de la barra lateral.
  2. Ve a github.com → busca el repo template experiences-template → clic en "Use this template" → nombre: cliente-slug
  3. Sube el config.json descargado al nuevo repo (reemplaza el existente).
  4. En el repo: Settings → Pages → Branch: main → Save. Listo en ~2 minutos.
  5. URL pública: https://usuario.github.io/slug
